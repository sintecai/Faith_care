Fix ONLY Dues Management + Monthly Contribution Reminders so the system can identify PAID vs UNPAID families and send reminders to UNPAID families (max 2 per month). Keep changes minimal.

- Do NOT redesign UI.

  (1) dues DB layer (add one table if missing),

========================
A) Add a simple payments ledger (required)
========================
Create a table if missing: family_payments
Columns:
- payment_id uuid pk default gen_random_uuid()
- church_id uuid not null
- family_id uuid not null
- period_year int not null
- period_month int not null        -- 1..12
- amount numeric not null default 0
- paid_at timestamptz not null default now()
- paid_by_staff_id uuid null
Constraints:
- unique(church_id, family_id, period_year, period_month)

Purpose:
This tells us if the family has paid for a given month.

========================
B) Define “current period” and unpaid
========================
- Current period = current date’s year/month (server local time).
- A family is “PAID” if a family_payments row exists for (current year, current month).
- Otherwise “UNPAID”.

Do NOT use next_due_date to decide paid/unpaid. Keep next_due_date for schedule display only.

========================

  - Paid (green) if paid for current month
  - Unpaid (red) if not paid for current month
- Add filter tabs/buttons at top:

Actions:

- Add “Mark Unpaid” only if you previously stored a paid row:
  - Deletes the family_payments row for current year/month (or marks void).

========================
