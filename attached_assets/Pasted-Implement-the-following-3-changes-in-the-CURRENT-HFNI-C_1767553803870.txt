2) AUTO-SEND ANNOUNCEMENTS AS SMS TO ALL MEMBERS
======================================================
Goal:
Whenever admin creates an Announcement and posts/publishes it, it should automatically send as an SMS to all eligible members.

Implementation requirements:

A) Define “posted/published”
- If your announcement flow already has a “Post” button or a “status” field (draft/published), then trigger SMS sending ONLY when status becomes “published”.
- If there is no status field, treat “Create Announcement” submission as “published” and trigger SMS after saving to DB.
- Prevent accidental duplicates: do not re-send if admin edits an existing announcement.

B) Member eligibility to receive SMS
- Member must be active (is_active = true).
- Member must have a valid phone number in E.164 format starting with “+”.
- If a consent flag exists (consent_sms / opt_in_sms), only send when consent is true.
- If no consent flag exists, send to all active members with a phone number (but keep code ready to respect consent later).

C) SMS content format (short, readable)
- SMS body should include:
  - Church/app name prefix e.g. “HFNI Connect:”
  - Announcement title (short)
  - A short snippet of the body (limit total SMS length to ~300 chars)
  - Optional: “Open the app for details.”
Example:
"HFNI Connect: {TITLE} - {BODY_SNIPPET} (Open the app for details)"

D) Twilio sending
- Use Twilio Messages API: client.messages.create({to, from, body})
- Require env var in Replit Secrets:
  - TWILIO_FROM_NUMBER (the Twilio SMS-capable number)
- Reuse existing Twilio client module (do NOT create duplicates).
- Send messages server-side only.

E) Logging + delivery tracking
- Log each SMS attempt to outbound_messages (existing table):
  - channel='sms'
  - template_key='announcement_broadcast:<announcement_id>'
  - to_address=member phone
  - status: queued -> sent/failed
  - error_message if failed
- Add a broadcast tracking table if needed (optional), but prefer minimal:
  - a field on announcements like sms_broadcast_sent_at timestamptz
  - or a separate table announcement_broadcasts(announcement_id, sent_at, total_sent, total_failed)
- Duplicate protection:
  - If sms_broadcast_sent_at is set, do not broadcast again for the same announcement_id.
  - Or check outbound_messages for template_key prefix announcement_broadcast:<announcement_id>.

F) Performance (200 members max)
- Use sequential sending or small batches (e.g., 10 at a time) to avoid timeouts.
- Provide admin feedback:
  - After posting, show a summary: “SMS broadcast queued/sent to X members, failed Y”.
  - If sending is queued asynchronously in-process, at least show “Broadcast started”.

G) Admin control toggle (recommended)
- Add a checkbox on the Announcement create form:
  “Send as SMS to all members”
  Default: ON
- If unchecked, do not send SMS, only create the announcement.

Acceptance:
- Creating/publishing an announcement triggers SMS broadcast to eligible members.
- No duplicate sends on edit.
- Outbound_messages shows sent/failed records for the broadcast.
- If TWILIO_FROM_NUMBER missing, show a clear admin error message on the announcement page.
